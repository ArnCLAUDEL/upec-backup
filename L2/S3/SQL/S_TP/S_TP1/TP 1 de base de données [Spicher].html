TP 1 de base de données

Quelques mots à propos de SQLite

SQLite est une libraire qui implémente un SGBD SQL. Cette API est notamment utilisée pour manipuler des bases de données en local pour des applications légères. Par exemple, firefox utilise SQLite pour la gestion des données comptes utilisateurs (par exemple, l'enregistrement des mots de passe).

Le programme sqlite3 permet d'interroger une base de données SQLite en ligne de commande. Son utilisation est extèmement simple. En supposant que le fichier db.sqlite contient une base de données, la ligne de commande suivante permet de lancer la boucle interactive : 

sqlite3 -column -header -nullvalue NULL db.sqlite

Les différentes options utilisées sont :

  • -column : afficher les résultats sous forme de tableau
  • -header : afficher les noms des colonnes dans les résultats
  • -nullvalue NULL : afficher NULL dans les cases vides

Le programme ainsi lancé propose un prompt sqlite> attendant une requête SQL.

Pour quitter le programme, on peut soit utiliser la commande .quit soit utiliser le raccourci clavier CTL+D.

Etant donné qu'il est plus pratique d'utiliser un fichier pour travailler, en supposant que votre fichier de requêtes SQL se nomme requete.sql, la ligne de commande suivante permet l'évaluation du fichier : 

sqlite3 -header -column -nullvalue NULL db.sqlite '.read requete.sql'

Préparation de la base pour le TP 1

Pour pouvoir travailler il faut avant tout créer une base de données (i.e., le fichier db.sqlite). Le fichier base.sql décrit ci-dessous contient les requêtes SQL permettant de créer les tables utilisées dans les TD 1 et 2, et dans le TP 1.

Après avoir créé un répertoire de travail et enregistré le fichier base.sql dans ce répertoire, executez en ligne de commande : 

sqlite3 -init base.sql

Le fichier est db.sqlite est alors créé.

base.sql
DROP TABLE IF EXISTS PC;
DROP TABLE IF EXISTS Portables;
DROP TABLE IF EXISTS Imprimantes;
DROP TABLE IF EXISTS Produit;
 
CREATE TABLE Produit
(fabricant varchar(2) NOT NULL,
 modele    int NOT NULL,
 type      varchar(11) NOT NULL,
 PRIMARY KEY (fabricant,modele)
);
 
CREATE TABLE PC
(modele     int NOT NULL,
 processeur varchar(2) NOT NULL,
 RAM        int NOT NULL,
 DD         int NOT NULL,
 DVDBR      varchar(8) NOT NULL,
 prix       int NOT NULL,
 PRIMARY KEY (modele)
);
 
CREATE TABLE Portables
(modele     int NOT NULL,
 processeur varchar(2) NOT NULL,
 RAM        int NOT NULL,
 DD         int NOT NULL,
 ecran      int NOT NULL,
 prix       int NOT NULL,
 PRIMARY KEY (modele)
);
 
CREATE TABLE Imprimantes
(modele     int NOT NULL,
 type       varchar(15) NOT NULL,
 couleur    varchar(3) NOT NULL,
 prix       int NOT NULL,
 PRIMARY KEY (modele)
);

.backup db.sqlite

Une fois le fichier db.sqlite créer, il est possible d'évaluer des requêtes contenues dans un fichier tp1.sql de la façon suivante :

tp1.sql
SELECT * FROM PC;
SELECT * FROM Produit;
SELECT * FROM Portables;
SELECT * FROM Imprimantes;
sqlite3 -header -column -nullvalue NULL db.sqlite '.read tp1.sql'
