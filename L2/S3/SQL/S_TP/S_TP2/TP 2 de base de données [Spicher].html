TP 2 de base de données

On vous rappelle que la procédure d'initialisation de la BD est donnée dans le TP 1.

Création de la base

Le code SQL suivant permet la création des tables dans votre base de données et les remplit. Utiliser la commande : 

sqlite3 -init base.sql
base.sql
DROP TABLE IF EXISTS prereq;
DROP TABLE IF EXISTS creneau;
DROP TABLE IF EXISTS tuteur;
DROP TABLE IF EXISTS choix_option;
DROP TABLE IF EXISTS etudiant;
DROP TABLE IF EXISTS resp_module;
DROP TABLE IF EXISTS filiere;
DROP TABLE IF EXISTS enseignant;
DROP TABLE IF EXISTS cours;
DROP TABLE IF EXISTS departement;
DROP TABLE IF EXISTS salle_cours ;
 
CREATE TABLE salle_cours
(batiment varchar(15),
 no_salle varchar(7),
 capacite int,
 PRIMARY KEY (batiment, no_salle)
);
 
CREATE TABLE departement
(nom_dept varchar(20), 
 batiment varchar(15), 
 budget	  numeric(12,2) CHECK (budget > 0),
 PRIMARY KEY (nom_dept)
);
 
CREATE TABLE cours
(id_cours varchar(8), 
 intitule varchar(50), 
 nom_dept varchar(20),
 credits  int CHECK (credits > 0),
 PRIMARY KEY (id_cours),
 FOREIGN KEY (nom_dept) REFERENCES departement(nom_dept) ON DELETE SET NULL
);
 
CREATE TABLE enseignant
(ID        varchar(5), 
 nom       varchar(20) NOT NULL, 
 nom_dept  varchar(20), 
 salaire   numeric(8,2) CHECK (salaire > 29000),
 PRIMARY KEY (ID),
 FOREIGN KEY (nom_dept) REFERENCES departement(nom_dept) ON DELETE SET NULL
);
 
CREATE TABLE filiere
(id_cours   varchar(8), 
 id_filiere varchar(8),
 semestre   varchar(6) CHECK (semestre IN ('Fall', 'Winter', 'Spring', 'Summer')), 
 annee      int CHECK (annee > 1701 AND annee < 2100), 
 batiment   varchar(15),
 no_salle   varchar(7),
 id_creneau varchar(4),
 PRIMARY KEY (id_cours, id_filiere, semestre, annee),
 FOREIGN KEY (id_cours) REFERENCES cours(id_cours) ON DELETE cascade,
 FOREIGN KEY (batiment, no_salle) REFERENCES salle_cours(batiment, no_salle) ON DELETE SET NULL
);
 
CREATE TABLE resp_module
(ID         varchar(5), 
 id_cours   varchar(8),
 id_filiere varchar(8), 
 semestre   varchar(6),
 annee      int,
 PRIMARY KEY (ID, id_cours, id_filiere, semestre, annee),
 FOREIGN KEY (id_cours,id_filiere, semestre, annee) REFERENCES filiere(id_cours, id_filiere, semestre, annee) ON DELETE cascade,
 FOREIGN KEY (ID) REFERENCES enseignant(ID) ON DELETE cascade
);
 
CREATE TABLE etudiant
(ID       varchar(5), 
 nom      varchar(20) NOT NULL, 
 nom_dept varchar(20), 
 tot_cred int CHECK (tot_cred >= 0),
 PRIMARY KEY (ID),
 FOREIGN KEY (nom_dept) REFERENCES departement(nom_dept) ON DELETE SET NULL
);
 
CREATE TABLE choix_option
(ID          varchar(5), 
 id_cours    varchar(8),
 id_filiere  varchar(8), 
 semestre    varchar(6),
 annee       int,
 note        varchar(2),
 PRIMARY KEY (ID, id_cours, id_filiere, semestre, annee),
 FOREIGN KEY (id_cours,id_filiere, semestre, annee) REFERENCES filiere(id_cours,id_filiere, semestre, annee) ON DELETE cascade,
 FOREIGN KEY (ID) REFERENCES etudiant(ID) ON DELETE cascade
);
 
CREATE TABLE tuteur
(ID_etud varchar(5),
 ID_ens  varchar(5),
 PRIMARY KEY (ID_etud),
 FOREIGN KEY (ID_ens) REFERENCES enseignant (ID) ON DELETE SET NULL,
 FOREIGN KEY (ID_etud) REFERENCES etudiant (ID) ON DELETE cascade
);
 
CREATE TABLE creneau
(id_creneau	varchar(4),
 jour      varchar(1),
 hr_debut  int CHECK (hr_debut >= 0 AND hr_debut < 24),
 min_debut int CHECK (min_debut >= 0 AND min_debut < 60),
 hr_fin    int CHECK (hr_fin >= 0 AND hr_fin < 24),
 min_fin   int CHECK (min_fin >= 0 AND min_fin < 60),
 PRIMARY KEY (id_creneau, jour, hr_debut, min_debut)
);
 
CREATE TABLE prereq
(id_cours  varchar(8), 
 prereq_id varchar(8),
 PRIMARY KEY (id_cours, prereq_id),
 FOREIGN KEY (id_cours) REFERENCES cours(id_cours) ON DELETE cascade,
 FOREIGN KEY (prereq_id) REFERENCES cours(id_cours)
);

Requêtes SQL

Description rapide de la base

  • departement(nom_dept, batiment, budget) : les départements d’une université (américaine)
  • cours(id_cours, intitule, nom_dept, credits) : les intitulés des cours, avec leur nombre d’ECTS et le département dont ils dépendent
  • enseignant(ID, nom, nom_dept, salaire) : les enseignants, avec leur no. d’identifiant
  • salle_cours(batiment, no_salle, capacite) : la table contenant les salles de cours avec leur caractéristiques
  • filiere(id_cours, id_filiere, semestre, annee, batiment, no_salle, id_creneau) : la liste des filières de l’université, avec tout leur emploi du temps codé dans les id_creneau
  • resp_module(ID, id_cours, id_filiere, semestre, annee) : les responsables de modules, pour chaque année
  • etudiant(ID, nom, nom_dept, tot_cred) : la liste de tous les étudiants (passés et présents) de cette université
  • choix_option(ID, id_cours, id_filiere, semestre, annee, note) : les choix des modules que les étudiants ont fait pendant leur cursus
  • tuteur(ID_etud, ID_ens) : les tuteurs de chaque étudiant
  • creneau(id_creneau, jour, hr_debut, min_debut, hr_fin, min_fin) : la table détaillant les créneaux horaires des cours
  • prereq(id_cours, prereq_id) : les prérequis (dépendances entre les cours)

Requêtes SQL

  1. Trouver tous les enseignants du département “Biology”
  2. Trouver les noms des cours de 3 ECTS du département d’informatique (“Comp. Sci.”)
  3. Afficher les identifiants et intitulés des cours choisis de l'étudiant dont le ID est 12345
  4. Afficher le nombre total de crédits pour ce même étudiant, sans utiliser le champ tot_cred de la table etudiant
  5. Afficher pour chaque étudiant le total de crédits sans utiliser le champ tot_cred
  6. Afficher le nom des étudiants pour lesquels la valeur de tot_cred est correcte
  7. Trouver les noms des étudiants qui ont choisi un cours du département d’informatique (“Comp. Sci.”), sans doublon
  8. Trouver les noms des étudiants ayant suivi un cours qui n'est pas de leur département
  9. Donner pour chaque étudiant le nombre de cours qu'il a suivi hors de son département
  10. Donner la moyenne du nombre de cours pris hors département
  11. Trouver les ID des enseignants qui n’ont jamais été responsable de module
  12. Comme pour le point précédent, mais en affichant aussi les noms des enseignants
  13. Afficher pour chaque enseignant son nom et le nombre de modules dont il est responsable (attention de bien afficher 0 pour les enseignants n'ayant jamais été responsable)
  14. Trouver pour chaque filière le nombre maximal et minimal d’étudiants inscrits au cours d'un semestre (on ne se soucie pas des filières qui n’ont pas eu d’étudiants)
  15. Même question en affichant 0 pour les filières qui n'ont pas eu d'étudiants
  16. Trouver les cours dont l'identifiant commence par “CS-1”
  17. Donner le nom et le département de tous les enseignants qui ont été responsables des cours trouvés au point précédent
  18. Donner pour chaque salle de cours le nombre total d'heures de cours dispensées dans cette salle
  19. Donner pour chaque étudiant le nombre d'heures de cours suivi en 2009
  20. Donner pour chaque étudiant le cours pour lequel il a suivi le plus de nombre d'heures

Passage à l'échelle

Après avoir fait vos tests avec sur les quelques données précédentes, vous pouvez essayer de les appliquer sur la table suivante bien plus conséquente pour observer les délais dus aux calculs.
