TP 6

Stratégie de passage des arguments

La stratégie de passage des arguments en C est dite par valeur : la fonction reçoit une valeur pour chacun de ses arguments. Ainsi, dans l'appel f(x) seul le contenu de la variable x est transmis à f, et non la variable x elle-même. 

#include <stdlib.h>
#include <stdio.h>
 
void swap(int a, int b) {
  int c;
  printf("Before: a = %d, b = %d\n", a, b);
  c = a; a = b; b = c;
  printf("After: a = %d, b = %d\n", a, b);
}
 
int main() {
  int x = 123;
  int y = 456;
  printf("Before: x = %d, y = %d\n", x, y);
  swap(x, y);
  printf("After: x = %d, y = %d\n", x, y);
  return EXIT_SUCCESS;
}

  • Modifiez le code précédent en utilisant les pointeurs avec leurs opérateur de référencement (&) et de déréférencement (*) pour obtenir un comportement équivalent à un appel par référence.

Arithmétique des pointeurs

On considère le programme suivant : 

int main() {
  char c; char* ptr_c = &c;
  int i; int* ptr_i = &i;
 
  printf("prt_c = %p\n", ptr_c);
  printf("prt_i = %p\n", ptr_i);
 
  printf("prt_c = %p\n", ptr_c + 1);
  printf("prt_i = %p\n", ptr_i + 1);
 
  return EXIT_SUCCESS;
}

Argument réponse avec scanf

La fonction scanf est généralement utilisée pour récupérer une donnée entrée au clavier par un utilisateur. Son fonctionnement est identique à celui de printf où une chaîne de caractères est donnée en premier argument pour spécifier le format de l'entrée, et les arguments suivant correspondent aux variables qui devront être affectée par scanf.

A cause de la stratégie d'appel par valeur, l'écriture suivante ne peut pas fonctionner : 

int i;
printf("Veuillez entrer un entier : ");
scanf("%d", i);
printf("Vous avez entré %d\n", i);

Pour permettre à scanf de modifier le contenu de i, il suffit de lui passer l'adresse de la variable (sa taille étant obtenu par le qualifieur %d spécifiant qu'il s'agit d'une variable entière signée).

  • Testez le code précédent en passant à scanf l'adresse de i plutôt que son contenu
  • Faites varier cet exemple pour les qualifieurs %c, %u, %x, %f, %e et %p avec leurs types respectifs ; essayez pour chacun des entrées qui ne respectent pas le format spécifié.

Le qualifieur %s se comporte de façon différente des autres

  • pour printf : %s attend une chaîne de caractères, donc un tableau de caractères, donc un pointeur de type char*
  • pour scanf : %s attend l'adresse mémoire où stocker une chaîne de caractères, donc un tableau de caractères, donc un pointeur de type char*
char* s = "\0\0\0\0"; 
printf("Veuillez entrer une chaîne : ");
scanf("%4s", s);
printf("Vous avez entré %4s\n", s);
  • Testez le code précédent et faites varier les entrées.

Ragoût de pointeurs

int main() {                                          
  char * c[4] = {"ENTER","NEW","POINT","FIRST"};
  char ** cp[4] = { c+3, c+2, c+1, c};
  char *** cpp = cp;
 
  printf("%s",**++cpp);
  printf("%s ",*--*++cpp+3);
  printf("%s",*cpp[-2]+3);
  printf("%s\n",cpp[-1][-1]+1);
  return EXIT_SUCCESS;
}
